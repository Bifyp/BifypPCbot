# Отчет о проблемах стабильности

## Критические проблемы (50/50 работа функций)

### 1. **Camera (handlers/camera.py)**
**Проблема:** `cv2.VideoCapture(0)` часто не освобождает камеру
- Строка 30: камера захватывается, но если ошибка до `cap.release()` — камера остается занятой
- **Решение:** Использовать `with` или `try-finally`

### 2. **Screenshot Recording (handlers/screenshot.py)**
**Проблема:** Запись видео блокирует поток и может крашиться
- Строка 122-130: `cv2.VideoWriter` без обработки ошибок инициализации
- Временный файл может не удалиться при ошибке
- **Решение:** Добавить проверку `out.isOpened()` и гарантированную очистку

### 3. **Audio Recording (handlers/audio.py)**
**Проблема:** `sounddevice` блокирует при недоступном микрофоне
- Строка 31-35: `sd.rec()` + `sd.wait()` зависает если микрофон занят
- Нет таймаута на `sd.wait()`
- **Решение:** Добавить проверку доступности устройства и таймаут

### 4. **Terminal (handlers/terminal.py)**
**Проблема:** Кодировка cp866 не всегда корректна
- Строка 42: `encoding="cp866"` работает не для всех команд
- Некоторые программы выводят UTF-8
- **Решение:** Попробовать несколько кодировок или использовать `errors='replace'`

### 5. **Files (handlers/files.py)**
**Проблема:** Обрезка путей до 60 символов
- Строки 40, 44, 49: `[:60]` обрезает длинные пути
- Callback data теряет полный путь → файл не найдется
- **Решение:** Использовать хеш-маппинг или base64 для длинных путей

### 6. **Media Volume (handlers/media.py)**
**Проблема:** COM threading не всегда инициализируется
- Строка 41: `comtypes.CoInitialize()` может упасть если COM уже инициализирован
- **Решение:** Использовать `CoInitializeEx` с флагом `COINIT_MULTITHREADED`

### 7. **Web Server (web/server.py)**
**Проблема:** MJPEG stream может зависнуть
- Строка 177: `_prev_frame_hash` глобальная — при нескольких клиентах конфликт
- Строка 204: `ws.receive(timeout=30)` но нет reconnect логики
- **Решение:** Сделать hash per-client, добавить heartbeat

### 8. **System (handlers/system.py)**
**Проблема:** Wi-Fi парсинг зависит от локали
- Строка 144: парсинг вывода `netsh` на русском/английском
- На английской Windows не найдет "Сигнал"
- **Решение:** Парсить оба варианта или использовать XML формат

### 9. **Browser (handlers/browser.py)**
**Проблема:** `taskkill` убивает все окна браузера
- Строка 40-42: закрывает ВСЕ окна Chrome/Firefox/Edge
- Потеря несохраненных данных
- **Решение:** Предупреждение или graceful shutdown

### 10. **Bot Polling (bot.py)**
**Проблема:** Нет обработки сетевых ошибок
- Строка 195: `infinity_polling` упадет при потере сети
- **Решение:** Добавить `exception_handler` и retry логику

## Дополнительные проблемы

### Memory Leaks
- `_monitor_threads` в screenshot.py не очищается при ошибках
- `_current_dirs` в files.py растет бесконечно

### Race Conditions
- Глобальные переменные без locks (`_tunnel_url`, `_prev_frame_hash`)
- Множественные потоки пишут в один лог без синхронизации

### Error Handling
- Многие `except Exception` без логирования деталей
- Пользователь видит только "❌ Ошибка: {e}" без контекста

## Быстрые фиксы

Для немедленного улучшения стабильности:
- Добавить `try-finally` во все места с ресурсами (камера, файлы, COM)
- Заменить все `encoding="cp866"` на `encoding="cp866", errors="replace"`
- Увеличить все таймауты в 2 раза
- Добавить логирование полных traceback'ов
